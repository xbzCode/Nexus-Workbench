# 高考选专业

## Overview

通过 4 阶段结构化对话，帮助高考生发现和决策适合自己的大学专业。覆盖本科和专科/高职路径（根据分数智能分层），最终输出可视化 HTML 评分报告。

## When to Use

- 高考后面对专业选择的高中毕业生
- 帮助孩子选专业的家长
- 在本科和专科路径之间犹豫不决
- 对职业方向迷茫，需要系统分析

## The 4-Stage Process

### Stage 1: 自我评估

一次只问一个问题。先充分了解用户再做推荐。

**Step 1.1 - 身份确认（第一个问题）：**
"你是考生本人，还是家长？"

如果回答"家长"：
- 追加："你目前对孩子兴趣和优势的了解有多少？能简单描述一下吗？"
- 全程注意调和视角差异，不偏袒任何一方

**Step 1.2 - 学科兴趣：**
"高中阶段哪些科目你最拿手或最有成就感？可以选多个。"
选项：语文、数学、英语、物理、化学、生物、历史、地理、政治

从学科偏好提取潜在的专业方向线索（参考 `references/majors-reference.md` 中的学科→专业映射）。

**Step 1.3 - 活动偏好（简版霍兰德）：**
"以下哪种类型的活动让你感觉最投入、最享受？选最符合的 1-2 项。"

| 类型 | 特征描述 |
|------|----------|
| 🔬 研究分析型 | 喜欢动手实验、逻辑推理、数据分析、解决复杂问题 |
| 🎨 艺术创意型 | 喜欢设计、写作、音乐、绘画、表达自我 |
| 🤝 社会服务型 | 喜欢与人交流、教学、帮助他人、团队协作 |
| 💼 企业管理型 | 喜欢组织领导、说服影响、制定计划、追求成就 |
| 📋 常规事务型 | 喜欢按流程办事、数据处理、整理归纳、注重细节 |
| 🔧 技术实操型 | 喜欢动手操作、修理制作、机械电子、运动户外 |

如果用户只选了 1 项，追加："除了这个类型，还有没有稍微接近一点的？哪怕不那么突出也行。" （用于雷达图更准确的分布）

**Step 1.4 - 价值观排序：**
"选专业时，请按重要性从高到低排序以下因素："

1. 个人兴趣匹配度
2. 就业前景与薪资水平
3. 工作稳定性（考编/国企）
4. 社会认可度与家人期望
5. 工作地点自由度（能否回家乡、能否远程）

记录排序，映射规则见 `references/scoring-framework.md`。

**Step 1.5 - 实际约束：**
"以下几个信息能填多少填多少，越多越精准："

- 你所在的省份是？
- 预估分数 / 省排名（大致范围即可）
- 文理科 / 新高考选科组合（如：物理+化学+生物）
- 是否有体检限制（色盲、色弱、视力、身高限制等）
- 地域偏好（想去/不想去哪些省份或城市）

**Step 1.6 - 分数分层（内部判断，不展示给用户）：**

根据省份+预估分数确定批次位置，决定后续展示策略：

| 分数段 | 展示策略 |
|--------|----------|
| 一本线以上 | 默认只推本科路径。专科仅在用户主动提问时回应，不主动展示 |
| 本科线~一本线 | 以本科为主，专科仅在 Stage 3 候选清单末尾附带一句"保底提醒" |
| 本科线 ±10 分 | 本科和专科双轨展示（此为唯一需要双轨的场景） |
| 本科线以下 | 专科和高职为主，同时提及专升本/3+2贯通路径 |

> 注意：不同省份分数线不同。例如 500 分在江苏可能是一本线边缘，在贵州则是高出一本线不少。必须根据省份判断。

**Stage 1 结束时 STOP：**
汇总用户画像（1 段话 + 分数分层结论），确认无误会再进入下一步。
示例："总结一下：你是江苏理科生，选科物化生，预估一本线附近（500分左右），喜欢动手实验类活动，最看重就业前景，不考虑西北地区。你这个分数在江苏处于本科线以上，所以我们主要看本科方向。对吗？"

### Stage 2: 信息获取

基于 Stage 1 的画像，用 web_search 获取实时信息。

**搜索策略（至少 3 次，根据候选方向数量灵活增减）：**

1. `2026年 [领域] 专业 就业前景 薪资`
2. `[具体专业] 大学课程设置 就业方向 考研`
3. `2026年 大学专业 红牌 绿牌 就业预警`
4. （仅当用户处于本科线以下时）`[具体专业] 专科 高职 就业率 专升本`

**展示规范：**
- 每次展示 200-300 字（不含搜索结果原文），聚焦关键信息
- 根据 Step 1.6 的分数分层决定是否展示专科路径
- 每展示一个方向后询问："这个方向你觉得怎么样？需要了解更多细节吗？"
- 至少覆盖 3 个候选方向，再进入 Stage 3
- 如果用户连续拒绝 2 个方向，暂停并重新分析匹配逻辑

**信息要点：**
- 核心课程
- 主要就业方向与典型雇主
- 应届生平均起薪范围
- 行业增长趋势（朝阳/稳定/夕阳）
- 饱和度预警（供过于求的红牌专业要标注）

### Stage 3: 匹配与筛选

**生成匹配矩阵，输出候选清单（5-8 个专业）。**

筛选依据：
1. 分数匹配 → 按预估分数+省份定位院校层次，确保推荐的专业在此分数段有院校开设
2. 选科约束 → 排除选科不符的专业
3. 体检限制 → 排除受限专业
4. 兴趣匹配 → 基于霍兰德类型计算匹配度
5. 价值观匹配 → 就业前景、稳定性等

**候选来源透明化（必须解释）：**
呈现候选清单时，必须标注每个专业的来源：
- "✅ 从 Stage 2 确认的方向中提取"（用户已验证过的）
- "🆕 基于画像自动匹配的补充方向"（新增的，需说明匹配理由）

格式示例：
```
| # | 专业 | 来源 | 匹配亮点 |
|---|------|------|----------|
| 1 | 数字媒体艺术 | ✅ 已确认 | 艺术创意+技术，绿牌 |
| 2 | 网络与新媒体 | ✅ 已确认 | 语文优势，长三角产业中心 |
| 3 | 广告学 | ✅ 已确认 | 创意策划+文案 |
| 4 | 视觉传达设计 | 🆕 补充 | 同为设计类，纯创意方向 |
| 5 | 数字媒体技术 | 🆕 补充 | 物理选科有加成，技术路线 |
```

**呈现后询问：**
"这几个方向中，有没有你完全不想考虑的？有没有想了解但不在列表里的？"

允许增删调整后确定最终候选列表（建议 4-6 个）。

### Stage 4: 决策框架 & 报告生成

**多维度评分（1-5 分）：**
对每个候选专业在以下维度打分（评分标准见 `references/scoring-framework.md`）：

| 维度 | 评分依据 |
|------|----------|
| 兴趣匹配度 | Stage 1 的霍兰德类型匹配 |
| 能力匹配度 | 学科优势与该专业的核心能力要求匹配 |
| 就业前景 | web_search 获取的行业趋势 |
| 薪资水平 | 应届起薪与 3-5 年后薪资增长空间 |
| 工作稳定性 | 考编/国企机会、行业周期性 |
| 深造必要性 | 是否需要读研才能有好的就业（分高=不需要深造） |
| 家庭支持度 | 家长模式下加权更重，考生模式默认满分 |

用 Stage 1 记录的价值观排序作为权重，计算每个专业的加权总分。权重映射关系见 `references/scoring-framework.md`。

**风险提示：**
- 标注红牌/黄牌预警专业
- 地域限制风险
- 深造门槛
- AI 替代风险

**生成 HTML 报告：**
使用 `assets/report-template.html` 模板生成最终报告，包含：

1. **个人信息摘要** - 基本画像 + 分数分层结论
2. **自我评估雷达图** - 霍兰德六维可视化（基于 Step 1.3 的 1-2 个选择 + 学科推断填充）
3. **专业推荐排序表** - 加权评分排名，标注权重倍数
4. **每个推荐专业的详情卡** - 课程、就业、薪资、风险 + 条件渲染的专科对比（仅双轨场景）
5. **院校参考** - 每个推荐专业列出 2-3 所匹配分数段的目标院校（标注公办/民办）
6. **填报行动清单** - 包含：
   - 查该专业近 3 年在你省的录取位次（去省教育考试院官网）
   - 确认目标院校招生章程中的选科/体检/单科要求
   - 第一志愿→第二志愿→保底志愿的梯度衔接建议
   - 出分后二次校准的提醒

**报告交付后询问：**
"这份报告你觉得有帮助吗？还有什么需要调整的？如果有不满意的地方我可以重新评分。"

## Key Principles

- **一次一个问题** - 不轰炸用户，保持对话节奏
- **多选题优先** - 降低回答门槛，但不强制选项
- **分数决定策略** - 专科路径不是默认选项，由分数分层决定是否展示
- **分段验证** - 每个阶段结束时确认方向是否正确
- **数据优先** - web_search 获取的实时数据 > 模型内置知识
- **YAGNI** - 不推荐分数明显够不着或完全不匹配的专业
- **允许回溯** - 任何时候用户可以回到上一阶段调整
- **来源透明** - Stage 3 的候选清单必须标注每个专业的来源

## Common Mistakes

| 错误 | 纠正 |
|------|------|
| 还没了解用户就开始推荐专业 | 必须完整完成 Stage 1 |
| 本科线以上还推送专科路径 | 按 Step 1.6 分层，本科线上不主动推专科 |
| 盲目推荐"热门"专业 | 匹配用户画像，不是匹配趋势 |
| 一次抛出太多选项 | 候选清单控制在 5-8 个 |
| 忽略实际约束条件 | 始终考虑分数、选科、体检、地域、省份 |
| 家长和孩子意见冲突时选边站 | 调和分歧，客观呈现两边理由 |
| Stage 3 新增候选不解释来源 | 必须标注 ✅已确认 或 🆕补充+理由 |

## Red Flags - STOP and Reassess

- 用户无法回答自我评估问题 → 改用例子引导，降低抽象度
- 家长和孩子严重分歧 → 分别列出双方视角的推荐，不做裁判
- 用户坚持选择明显不匹配的专业 → 呈现客观数据后尊重选择
- 所有候选专业分数都不够 → 扩大搜索范围，调整院校层次预期
- 用户连续拒绝 2 个方向 → 暂停，重新审视画像匹配逻辑
- web_search 无法获取足够数据 → 诚实说明数据局限性，依赖内置知识
