# 简历生成器

生成现代简约风格的中文简历，输出为自包含 HTML 文件，支持浏览器打印与一键导出 PDF。

## 模板

| 模板 | 布局 | 风格 | 适用场景 |
|------|------|------|----------|
| `modern` | 单栏 | 简约留白，线条感 | 通用，互联网/科技行业 |
| `professional` | 双栏（左侧边栏+右主区域） | 深色侧边栏，信息密度高 | 传统行业、咨询、金融 |

默认使用 `modern`，通过 `--template <name>` 切换。

## 用户输入方式

支持两种数据输入，**对话优先**：

1. **对话输入（默认）**：用户口述或分段描述，AI 整理为结构化数据后填入模板
2. **结构化文件输入**：用户提供 JSON/YAML 文件，按 [data-schema.md](references/data-schema.md) 格式

当用户提供非结构化信息时，AI 应：
- 提取关键信息并映射到简历各板块
- 对缺失的必要信息主动追问（一次最多问 3 个问题）
- 将整理好的数据以 JSON 格式暂存，供用户确认后再生成

## 选项

| 选项 | 说明 |
|------|------|
| `--template <name>` | 模板：modern（默认）、professional |
| `--accent <name>` | 主题色：slate（默认）、blue、emerald、violet、rose、amber |
| `--no-photo` | 不显示照片占位区域 |
| `--quick` | 跳过确认，使用默认选项直接生成 |
| `--compact` | 紧凑模式，减少间距以容纳更多内容 |
| `--data <file>` | 从 JSON/YAML 文件读取简历数据 |

## 主题色方案

| 主题色 | 主色值 | 适用场景 |
|--------|--------|----------|
| slate | `#475569` | 通用、沉稳 |
| blue | `#2563eb` | 科技、互联网 |
| emerald | `#059669` | 环保、教育、医疗 |
| violet | `#7c3aed` | 创意、设计 |
| rose | `#e11d48` | 时尚、新媒体 |
| amber | `#d97706` | 金融、咨询 |

## 简历板块

完整板块参考 [sections.md](references/sections.md)。核心板块：

| 板块 | 必要 | 说明 |
|------|------|------|
| 头部信息 | ✅ | 姓名、联系方式、求职意向 |
| 工作经历 | ✅ | 公司、职位、时间、职责描述 |
| 教育背景 | ✅ | 学校、专业、学历、时间 |
| 专业技能 | ⬜ | 技能分类与熟练度 |
| 项目经验 | ⬜ | 项目名称、角色、描述、成果 |
| 自我评价 | ⬜ | 简短个人总结 |
| 证书/荣誉 | ⬜ | 证书名称、颁发机构、时间 |
| 语言能力 | ⬜ | 语言及水平 |

## 工作流

### 进度检查表

```
简历生成进度：
- [ ] Step 1: 收集简历数据 ⛔ BLOCKING
- [ ] Step 2: 确认选项（模板、主题色、板块、紧凑模式） ⚠️ 除非 --quick
- [ ] Step 3: 生成 HTML 文件
- [ ] Step 4: 完成报告
```

### 流程

```
输入 → [Step 1: 收集数据] ─┬─ 对话 → 提取+追问 → 确认数据
                            └─ 文件 → 解析+校验 → 补充追问
        ↓
      [Step 2: 确认选项] ── 主题色 / 板块 / 紧凑模式
        ↓                    (跳过如果 --quick)
      [Step 3: 生成 HTML] ── 基于模板 + 数据 → 自包含HTML
        ↓
      [Step 4: 完成报告] ── 输出文件路径 + 使用提示
```

### Step 1: 收集简历数据 ⛔ BLOCKING

**对话输入**时：
1. 从用户消息中提取所有简历相关信息
2. 将信息映射到 [data-schema.md](references/data-schema.md) 的结构
3. 检查必要板块是否完整（头部信息 + 至少一项：工作经历 或 教育背景）
4. 缺失必要信息 → 追问（一次最多 3 个问题，避免反复）
5. 将整理后的 JSON 数据展示给用户确认

**文件输入**时（`--data <file>`）：
1. 读取并解析 JSON/YAML 文件
2. 按 [data-schema.md](references/data-schema.md) 校验格式
3. 报告缺失字段，追问补充

### Step 2: 确认选项 ⚠️

除非用户指定 `--quick` 或在消息中明确说"直接生成"、"不用确认"等，否则必须确认：

1. **模板**：modern（简约单栏）/ professional（专业双栏），默认 modern
2. **主题色**：推荐默认 slate，展示选项列表
3. **板块选择**：列出所有可选板块，标记默认包含的
4. **紧凑模式**：内容较多时建议开启

### Step 3: 生成 HTML 文件

1. 根据 `--template` 选择对应模板：
   - `modern` → `assets/template-modern.html`
   - `professional` → `assets/template-professional.html`
2. 将确认后的数据填入模板对应位置
3. 根据选项调整：主题色、板块显隐、间距模式
4. 确保自包含：内联 CSS、内联字体、html2canvas + jsPDF CDN 引用
5. 输出到用户当前工作目录或指定路径

**modern 模板定制点**：
- `<!-- ACCENT_COLOR -->`：替换主题色 CSS 变量
- `<!-- RESUME_CONTENT -->`：替换简历主体内容
- `<!-- PHOTO_SECTION -->`：照片区域（`--no-photo` 时移除）
- `<!-- COMPACT_MODE -->`：紧凑模式 CSS 类

**professional 模板定制点**：
- `:root { --accent: ...; --sidebar-bg: ...; }`：替换主题色
- 侧边栏内容：照片、联系方式、技能、语言、自我评价
- 主区域内容：工作经历、项目经验、教育背景、证书

### Step 4: 完成报告

```
简历已生成！

姓名：[姓名]
模板：[modern/professional] | 主题色：[accent] | 紧凑模式：[on/off]
板块：[包含的板块列表]
语言：中文

文件：
✓ [文件名].html

使用方式：
- 浏览器打开查看 → 双击文件即可
- 🔑 推荐 Ctrl+P → "另存为PDF"（生成可编辑文字的PDF）
- 一键导出PDF → 点击页面右上角"导出PDF"按钮（生成图片型PDF）
```

## 文件结构

输出目录：当前工作目录（或用户指定路径）

```
<output-dir>/
└── resume-[姓名]-[日期].html    # 自包含HTML简历文件
```

## PDF 导出

参考 [pdf-export.md](references/pdf-export.md)。两种方式：

1. **浏览器打印**（默认推荐）
   - 打开 HTML → Ctrl+P → 目标选"另存为PDF"
   - 零依赖，效果稳定，兼容所有浏览器

2. **一键导出**（html2canvas + jsPDF）
   - 页面右上角内置"导出PDF"按钮
   - 点击后自动生成并下载 PDF
   - 基于 html2canvas + jsPDF CDN，需联网首次加载
   - 已解决：白色蒙层、文字模糊、空白页等已知问题

## 输出要求

始终生成单个自包含 `.html` 文件：
- 嵌入所有 CSS（无外部样式表，除 Google Fonts）
- 引入 html2canvas + jsPDF CDN（用于一键导出功能）
- 无需额外 JavaScript 文件
- 文件在任意现代浏览器中直接打开即可正确渲染
- 打印时自动隐藏操作按钮，优化为 A4 页面布局

## 修改与重生成

| 操作 | 步骤 |
|------|------|
| **修改内容** | 告知要修改的字段 → 更新数据 → 重新生成 HTML |
| **更换主题色** | 确认新颜色 → 替换 CSS 变量 → 重新生成 |
| **增删板块** | 确认板块变更 → 调整模板 → 重新生成 |
