# 需求细化 Skill

此节点的作用是在执行任务前，强制与用户进行一轮需求细化交互。

## 工作流程

1. **接收原始需求**：用户输入一个粗略的请求，如"帮我生成架构图"
2. **主动提问**：Agent 必须先向用户提出至少一个包含"？"的问题，明确需求细节
3. **等待用户回答**：Agent 暂停，等待用户通过审批 API 回答问题
4. **结合回答执行**：收到用户回答后，将原始需求与补充信息合并执行

## 使用场景

- 用户需求模糊，需要进一步明确
- 测试交互机制（QuestionDetectedEvent → approval → resume）
- 确保任务执行前有充分的需求理解

## 示例对话

**用户**: 帮我生成架构图
**Agent**: 请问您需要什么场景的架构图？是微前端、微服务、还是云基础设施？
**用户**: 微前端的架构图
**Agent**: 返回关于微前端架构图的详细描述
